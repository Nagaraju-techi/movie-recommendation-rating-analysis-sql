# 🎬 Movie Recommendation & Rating Analysis System

## 📌 Overview

The **Movie Recommendation & Rating Analysis System** is a SQL-based mini project designed to simulate how streaming platforms manage user activity, ratings, and movie insights.

This project focuses on storing structured movie data and analyzing user behavior using SQL queries to generate meaningful recommendations and trends. It demonstrates core database concepts such as schema design, relationships, joins, aggregations, and analytical reporting.

Developed using **MySQL Workbench**.

---

## 🚀 Objectives

- Design a relational database for movie analytics
- Store user profiles, movie catalog, ratings, and watch history
- Analyze rating patterns and user engagement
- Generate recommendation-ready insights
- Identify trending content and popular genres

---

## 🛠️ Tech Stack

- **Database:** MySQL  
- **Tool:** MySQL Workbench  
- **Language:** SQL  

---

## 🗃️ Database Structure

The system consists of the following tables:

- **Users** – Stores user details  
- **Movies** – Stores movie information and genres  
- **Ratings** – Stores ratings provided by users  
- **Watch_History** – Tracks movie watch activity  

These tables are connected using primary and foreign key relationships.

---

## ✨ Key Features

- ⭐ Top-rated movie analysis  
- 🎭 Genre popularity insights  
- 🎯 Basic recommendation logic using ratings  
- 👤 User watch behavior analysis  
- 📈 Trending movies detection  
- 🔗 Relational database design with constraints

## 📊 Business Insights Generated

This project can help answer questions such as:

- Which genres are watched the most?
